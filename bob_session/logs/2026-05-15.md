## Challenges

### Challenge 1: Vector Storage in SQLite
**Issue**: SQLite doesn't natively support vector operations efficiently
**Solutions Considered**:
1. Use sqlite-vec extension (simple, local)
2. Use ChromaDB (separate service, more features)
3. Use Postgres with pgvector (production-ready, requires migration)

**Decision**: Start with in-memory cosine similarity for MVP, migrate to ChromaDB or pgvector if needed

### Challenge 2: Embedding Generation
**Issue**: Balance between cost, speed, and quality
**Options**:
1. OpenAI API ($$$, best quality, cloud dependency)
2. @xenova/transformers (free, local, good quality)
3. Ollama embeddings (free, local, requires setup)

**Decision**: Use @xenova/transformers for MVP (free, no API keys), add OpenAI as optional upgrade

### Challenge 3: LLM Choice
**Issue**: Cost vs. quality vs. privacy
**Options**:
1. OpenAI GPT-4 (best quality, $$$, cloud)
2. Ollama Llama 3 (free, local, good quality)
3. Claude API (good quality, $$, cloud)

**Decision**: Support both OpenAI and Ollama, let users choose based on needs
